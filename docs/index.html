Argos

Spec-driven orchestration for AI coding agents.

You queue a ticket. Argos plans it, writes the code and tests in an isolated git worktree, checks the result against the ticket's acceptance criteria, and pushes a branch. It never merges; that part is yours.

It works as a scaffold for Claude Code, Cursor, Codex CLI, and Gemini CLI, and ships a Python CLI (stdlib only, no runtime dependencies) that runs the whole loop headlessly. Watch a real session below, browse the live spec graph, or read the source.

The loop

argos/specs/          PRD, ARCHITECTURE, STATE.md, tickets, ADRs
     |
     |  argos queue add TICKET-ID
     v
orchestrator          picks independent tickets, one git worktree each
     |
     v
planner -> coder -> watchdog -> verifier
                                   |
                                   v
                             branch pushed
                              (you merge)

A session ends one of three ways:

  • Pass. The verifier pushes a branch and appends to STATE.md. You review and merge; Argos never merges on its own.
  • CHAOS_BLOCKED. The watchdog caught the coder drifting from the plan. The loop halts and waits for /steer. A trivially flagged plan retries once, planner to coder; anything structural goes to you.
  • Failed verify. A failed ticket. The verifier never retries; you decide what happens next.

Run it on your own repo

Argos works on any codebase. Your repo declares its language, dependencies, and test command in argos/conventions.md, and every dispatched session builds to those rules.

git clone https://github.com/kingl0w/argos.git
cd your-repo
argos init            # scaffold argos/specs/, write conventions.md, install hooks
$EDITOR argos/conventions.md   # your language, deps, test command
argos queue add ABC-001        # queue a ticket you wrote under argos/specs/
argos orchestrate     # builds, tests, verifies, pushes a branch. you merge

Why it's built this way

Specs are the source of truth.

Markdown specs (PRD, architecture, tickets, ADRs, STATE.md) drive every session. Hooks keep them honest: STATE.md is append-only, and argos status fails when the specs contradict each other. The spec graph is projected straight from those files.

Five agents, each with a narrow job.

The planner never writes code. The coder can't touch specs. The watchdog diffs the code against the plan and halts on drift. Only the verifier runs tests and updates state. Narrow tool permissions make the loop auditable instead of hopeful.

The human owns the merge.

Sessions run in parallel worktrees when independence detection clears them, and every session ends the same way: a pushed branch. Ambiguity escalates to the operator. There is no silent recovery.
